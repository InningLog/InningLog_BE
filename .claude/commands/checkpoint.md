# /checkpoint - 작업 체크포인트 갱신

세션 종료 전 현재 작업 상태를 `docs/checkpoint.md`에 기록합니다.

## 실행 절차

1. **현재 상태 파악**
   - `git status` - 커밋 안 된 변경사항 확인
   - `git branch` - 현재 브랜치 확인
   - `gh pr list` - 열린 PR 확인

2. **docs/checkpoint.md 갱신**
   - 마지막 업데이트 날짜 수정
   - 현재 진행 중인 작업 업데이트
   - 완료된 항목 체크 [x]
   - 새로운 TODO 추가
   - 발견된 이슈 기록

3. **커밋 (선택)**
   - 사용자에게 커밋 여부 확인
   - `docs : 체크포인트 갱신` 으로 커밋

## 체크포인트 파일 구조

```markdown
# 작업 체크포인트
> 마지막 업데이트: YYYY-MM-DD

## 현재 진행 중인 작업
- 이슈/PR 번호
- 브랜치명
- 상태 (진행중/PR대기/완료)
- 완료된 작업 체크리스트

## 다음 세션 TODO
- [ ] 할 일 목록

## 발견된 이슈
- 이슈 설명 및 해결 방안

## 관련 문서
- 문서 경로 링크
```

## 사용 예시

```
사용자: /checkpoint
Claude: 현재 작업 상태를 확인하고 체크포인트를 갱신합니다...
        [체크포인트 내용 요약]
        커밋할까요? (y/n)
```
