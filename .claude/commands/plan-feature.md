# 기능 기획서 작성

기능 추가를 위한 기획서를 작성하고 `docs/plan/` 폴더에 저장합니다.

## 사용법
```
/plan-feature <기능명>
```

예시:
- `/plan-feature journal-comment` - 직관일지 댓글 기능 기획
- `/plan-feature user-notification` - 알림 기능 기획

## 작업 절차

1. **코드 분석**: 관련 도메인의 현재 구조 파악
   - 엔티티, 서비스, 컨트롤러, DTO 구조 분석
   - 재사용 가능한 기존 코드 식별
   - 유사 기능이 있다면 해당 구현 참고

2. **API 설계**: RESTful 엔드포인트 설계
   - HTTP Method, URL 경로
   - Request/Response 스펙
   - 인증/인가 요구사항

3. **구현 계획**: 수정/생성 필요한 파일 목록
   - 엔티티 변경사항
   - 서비스 레이어 변경
   - 컨트롤러 추가
   - DTO 변경

4. **문서 저장**: `docs/plan/feature-plan-{기능명}.md` 파일로 저장

## 기획서 템플릿

```markdown
# 기능 기획서: {기능명}

> 작성일: {날짜}
> 관련 이슈: #{이슈번호}
> 상태: 기획 완료

## 1. 개요
{기능 설명}

## 2. 현재 구조 분석
{관련 코드 분석 결과}

## 3. API 설계
{엔드포인트, Request/Response 스펙}

## 4. 구현 계획
{수정 파일 목록 및 변경 내용}

## 5. 체크리스트
- [ ] 구현 항목들...
```

## 주의사항
- 기존 아키텍처 패턴(Controller → Facade → Service → Repository)을 따를 것
- 재사용 가능한 기존 코드를 최대한 활용
- PR 컨벤션에 맞는 커밋 타입 명시
